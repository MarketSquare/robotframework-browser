# dynamic-test-app

Express-based HTTP/HTTPS server that provides static test pages and a logging
API for the Robot Framework Browser library acceptance tests.

## Architecture

```
node/dynamic-test-app/
├── src/
│   └── server.ts          # Express server entry point
├── static/                # Served at /  (express.static)
│   ├── *.html             # Test pages
│   ├── frames/            # Frameset/iframe pages
│   └── test-app-logger.js # Client-side event capture (injected into every page)
└── dist/
    └── server.js          # Compiled output (generated by `inv create-test-app`)
```

## Logging

All log output is written as newline-delimited JSON to **stdout**.  The test
runner captures this per-process into:

```
atest/output/test-app/test-app-<port>.log
```

### Log entry schema

Every entry has a `timestamp` field (ISO 8601 UTC) plus an `event` field that
identifies the entry type.

#### Server events

Written directly by `src/server.ts`.

| `event`        | Additional fields |
|----------------|-------------------|
| `server_start` | `url` |
| `http`         | `method`, `url`, `status`, `contentLength`, `responseTimeMs` |

Examples:
```json
{"timestamp":"2025-07-01T12:00:00.000Z","event":"server_start","url":"http://localhost:7272"}
```

```json
{"timestamp":"2025-07-01T12:00:00.250Z","event":"http","method":"GET","url":"/dist/","status":200,"contentLength":"588","responseTimeMs":3.998}
```

#### Context events — `POST /api/log/context`

Sent by `atest/library/test_app_listener.py` at Robot Framework lifecycle
boundaries:

| `event`      | Additional fields |
|--------------|-------------------|
| `start_suite` | `id`, `name`, `pid` |
| `end_suite`   | `id`, `name`, `pid`, `status` |
| `start_test`  | `id`, `name`, `pid` |
| `end_test`    | `id`, `name`, `pid`, `status` |

`id` follows the Robot Framework structural ID format: `s1`, `s1-s2`, `s1-s2-t3`.

Example:
```json
{"timestamp":"2025-07-01T12:00:00.000Z","event":"start_test","id":"s1-s2-t1","name":"Open Browser","pid":12345}
```

#### Page events — `POST /api/log/event`

Sent by `static/test-app-logger.js` from the browser page via
`navigator.sendBeacon` (XHR fallback).

| `event`      | Additional fields |
|--------------|-------------------|
| `load`       | `url`, `title` |
| `popstate`   | `url`, `title` |
| `click`      | `url`, `tag`, `text` |
| `change`     | `url`, `tag`, `type`, `text`, `value` |
| `submit`     | `url`, `tag`, `id`, `action` |
| `dragstart`  | `url`, `tag`, `text` |
| `drop`       | `url`, `tag`, `text` |
| `hover`      | `url`, `tag`, `text` |

`text` is derived from `aria-label` → `textContent` → `value` → `id` → `name`,
trimmed and capped at 80 characters.

`tag` values are emitted in lowercase (for example `button`, `a`, `input`).

Example:
```json
{"timestamp":"2025-07-01T12:00:01.234Z","event":"click","url":"http://localhost:7272/index.html","tag":"button","text":"Click me"}
```

## Adding a new test page

1. Create the `.html` file under `static/`.
2. Add `<script src="/test-app-logger.js"></script>` before `</body>` (or
   before `</html>` if the file has no `<body>` element).

## Building

```bash
inv create-test-app
```

This compiles `src/server.ts` → `dist/server.js` via esbuild.
